# 备件管理表结构设计

> 来源：`01-详细设计文档1024.docx`。本文件按业务模块沉淀当前实现表结构，便于产品理解系统现状和后续项目落地差异评估。

## 表清单

| 表名 | 产品可读对象 | 字段数 | 关键字段/关联线索 |
|---|---|---:|---|
| `spare_parts_po` | 备件 po | 12 | id（hc 备品备件 po）；purchase_order_status（hc 采购单状态（新建））；pr（hc PR单编号） |
| `spare_parts_pr_detail` | 备件 pr detail | 24 | id（hc 备品备件 采购单据明细 表身）；order_id（单据 id）；code（hc 备件 编号） |
| `spare_parts_pr_detail_sap` | 备件 pr detail sap | 14 | id（hc 备品备件 采购申请 sap收货）；order_id（hc order 表头 id）；detail_id（hc detail 表身 id）；code（hc 备件 编号） |
| `spare_parts_pr_order` | 备件 pr order | 18 | id（hc 备品备件 pr 单据 order 表头）；status（hc 物料需求联络单=wms状态 (list字段)）；requestid（hc oa回传的 requestid） |
| `spare_parts_transfer_detail` | 备件 transfer detail | 16 | id（hc 备品备件 PMC调拨 明细 detail 表身）；order_id（单据 id）；code（hc 备件 编号） |
| `spare_parts_transfer_order` | 备件 transfer order | 21 | id（hc 备品备件 PMC调拨 单据 order 表头）；cbzx（hc SAP成本编号）；status（hc 单据状态 1新增(保存在本地) 2提交(传给OA) 3审核通过 4审核不通过）；requestid（hc oa回传的 requestid） |
