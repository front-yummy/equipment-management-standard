# 表结构设计索引

> 来源：`01-详细设计文档1024.docx` 的“数据库设计”章节。本文档集用于帮助产品理解当前系统实现中的业务对象、字段承载和模块边界；不直接等同于新标准产品的目标模型。

## 使用说明

- 每个模块文件包含“表清单”和“字段明细”。
- 字段明细保留原详细设计中的字段名、类型、长度、是否允许空值、默认值和说明。
- “产品可读对象/关键字段线索”是基于表名与字段说明的产品化解读，后续项目落地前仍需与研发和现场数据流核对。
- 工装管理来自详细设计文档，但当前需求分析目录未设置同名核心模块，因此作为实现补充单独列出。

## 模块业务解读

| 模块 | 当前实现中的核心业务对象 | 产品阅读重点 |
|---|---|---|
| OEE与KPI分析 | 设备 DT 目标、PPM、损失记录、损失类型、产能指标、快照与快照审计 | 重点关注 `oee_loss_record` 与维修工单、设备状态、责任部门/责任人的关联，以及快照表是否服务于看板固化、历史追溯和审计。 |
| 设备资产管理 | 验收记录、提醒设置、附件、设备 BOM、设备工厂/分组/责任人、项目、设备类型、故障字典、文件资料、设备台账 | 重点关注 `eq_information` 作为设备主数据的字段边界，以及项目表 `eq_equipment_project` 中新增、调拨、报废、闲置、升级改造等状态节点是否需要抽象成标准产品配置。 |
| 预防性维护-点巡检 | 点巡检基准、项目、计划、任务、报告、变更历史 | 重点关注“基准-项目-计划-任务-报告”的链路，任务表中同时保存设备、基准、计划、维修工单等外键，适合反向校验业务闭环。 |
| 预防性维护-设备保养 | 保养基准、项目、计划、任务、导入任务、报告、变更历史 | 结构与点巡检高度相似，后续标准产品可考虑统一抽象“预防性任务引擎”，再用类型区分点检、巡检、保养。 |
| 故障维修与异常工单 | 操作记录、委外工单、协助人、维修工单、维修工单备份、维修原因、停机工单 | 重点关注 `eq_repair_order` 的状态机、来源、OEE 回写、协助、委外、停机工单关联；`eq_repair_order_bk` 可能是历史备份表，产品侧不宜直接作为业务对象。 |
| 工装管理（详细设计来源） | 模具退模、报废、入库、主数据、机型、保养、型号、氮化、出库、返修、维修 | 该模块来自详细设计但不在当前核心需求目录中，建议作为可选扩展或独立子域评估，避免挤占设备管理 MVP 主流程。 |
| 备件管理 | PO、采购申请明细、SAP 收货、采购申请表头、PMC 调拨明细、PMC 调拨表头 | 重点关注采购申请、SAP 收货、PMC 调拨三条链路，以及字段中大量 `hc`、OA、SAP、WMS 语义是否属于客户现场集成，需要在标准产品中改为可配置集成字段。 |
