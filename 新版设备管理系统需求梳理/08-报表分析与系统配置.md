# 08. 报表分析与系统配置

## 1. 报表目标

报表不是只展示总数，而是要支持定位问题和发起动作。

## 2. 核心报表

| 报表 | 关键指标 |
|------|----------|
| 设备总览 | 设备数量、状态分布、健康等级 |
| 维护执行 | 点检完成率、保养达成率、逾期任务 |
| 异常闭环 | 异常数、闭环率、超期数、转维修率 |
| 维修分析 | 故障次数、MTTR、返修率、原因分布 |
| 备件分析 | 领用量、库存水位、关键备件风险 |
| 停机分析 | 停机时长、E10 分类、责任归因 |

## 3. 系统配置

| 配置 | 说明 |
|------|------|
| 权限角色 | 管理员、维修、保养、点检、备件、管理者 |
| 字典 | 设备等级、故障分类、异常等级、E10 状态 |
| 规则 | 任务生成、异常识别、预警升级、验收要求 |
| 模板 | 导入模板、标识牌模板、工单模板 |
| 集成 | MES、ERP、WMS、消息、数采 |

## 4. 待确认事项

### 4.1 管理层首页重点

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. OEE 优先 | 首页突出 OEE、停机、产出效率 | 管理价值高 | 数据来源要求高 |
| B. 维修/维护执行优先 | 首页突出待办、逾期、故障、MTTR、保养达成率 | 首版更容易落地 | 生产效率视角弱 |
| C. 角色化首页 | 管理者看指标，主管看工单，一线看待办 | 体验最好 | 需要角色配置 |

推荐：C. 角色化首页，首版默认维修/维护执行优先。

推荐原因：不同角色关注点不同。首版数据基础更适合先展示维护和维修闭环，OEE 后续增强。

### 4.2 数据权限隔离维度

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. 只按组织 | 用户看本部门数据 | 简单 | 跨部门设备责任难处理 |
| B. 按安装位置 | 用户看负责区域设备 | 符合工厂现场 | 需要位置授权 |
| C. 组织 + 位置 + 角色 | 管理组织、设备位置、角色能力共同决定权限 | 通用、安全 | 配置复杂 |

推荐：C. 组织 + 位置 + 角色。

推荐原因：设备管理常出现“设备属于生产，维修属于设备部”的跨组织协作。只按组织不够，必须叠加位置和角色。

### 4.3 多工厂多租户能力

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. 单工厂 | 只支持一个工厂 | 简单 | 集团客户不适用 |
| B. 多工厂单租户 | 一个企业下多个工厂，数据可隔离可汇总 | 符合多数制造集团 | 需要组织和位置层级 |
| C. 多租户 SaaS | 多企业租户隔离 | 商业化能力强 | 架构要求高 |

推荐：B. 多工厂单租户。

推荐原因：工业标品应支持集团多工厂汇总，但当前文档目标不是公网 SaaS 多租户。先做企业内多工厂更实用。
