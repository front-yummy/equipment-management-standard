# 新版设备管理系统需求梳理

本目录用于重新梳理一版更通用、更好用、可扩展的工业设备管理系统。它参考现有标品文档和外部文章方法论，但不按现有标品原样复刻。

## 设计原则

1. 先保证 MVP 主流程闭环，再扩展复杂能力。
2. 以设备对象为中心，而不是以表单数量为中心。
3. 台账只维护稳定主数据，业务数据来自流程和事件沉淀。
4. 点巡检、保养、维修、备件、指标必须互相联动。
5. 指标必须能下钻到动作，不能只做展示。
6. 复杂现场差异优先抽象为配置项、规则或扩展插件。
7. AI 先作为增强能力，不作为基础闭环的前置依赖。

## 当前建议的 MVP 范围

| 范围 | MVP 是否纳入 | 说明 |
|------|--------------|------|
| 设备台账与二维码 | 是 | 一机一档、批量导入、扫码查看/报修 |
| 设备类型与安装位置 | 是 | 类型决定基准，位置支持统计下钻 |
| 点巡检任务 | 是 | 支持基准、计划、扫码执行、异常记录 |
| 保养任务 | 是 | 支持计划自动生成、执行、验收可配置 |
| 维修工单 | 是 | 报修、派工、接单、处理、验收、关闭 |
| 备件领用 | 是 | 先支持维修/保养领用和履历 |
| OEE/MTTR/MTBF | 部分 | 先做维修指标和停机基础，OEE 可作为增强 |
| 健康评分 | 增强 | MVP 做健康摘要，不强依赖评分 |
| AI 推荐 | 增强 | 先沉淀知识库，再接推荐能力 |
