# 04. 备件与设备BOM

## 1. 模块目标

备件管理要解决两个问题：

1. 维修/保养时有没有备件可用。
2. 更换到设备上的备件后续能不能追踪寿命和风险。

## 2. 核心对象

| 对象 | 说明 |
|------|------|
| 备件主数据 | 备件编号、名称、规格、单位、供应商 |
| 库存 | 仓库、库位、现存量、安全库存 |
| 领用单 | 维修/保养领用 |
| 退库单 | 未使用或旧件退回 |
| 设备 BOM | 设备上的部件位置和可用备件 |
| 更换记录 | 旧件下线、新件上线 |

## 3. MVP 范围

| 能力 | 是否纳入 |
|------|----------|
| 备件台账 | 是 |
| 手工入库 | 是 |
| 维修/保养领用 | 是 |
| 设备更换履历 | 是 |
| 安全库存预警 | 增强 |
| 采购申请 | 增强 |
| 寿命预测 | 可选 |

## 4. 待确认事项

### 4.1 库存主系统

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. 本系统做库存主系统 | 入库、出库、库存都在设备系统维护 | 闭环完整 | 和 ERP/WMS 容易重复 |
| B. ERP/WMS 做库存主系统 | 本系统只调用库存和发起领用 | 口径统一 | 依赖外部系统 |
| C. 双模式 | 无 ERP/WMS 时本系统管库存，有外部系统时对接 | 通用性强 | 需要明确主数据和库存同步规则 |

推荐：C. 双模式。

推荐原因：标准产品要适配不同成熟度客户。小客户可本系统管理，大客户对接 ERP/WMS。

### 4.2 备件是否必须绑定 BOM 位置

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. 必须绑定 | 备件领用后必须绑定设备 BOM 位置才算在用 | 寿命、履历、追溯清晰 | 现场操作变重 |
| B. 不要求绑定 | 只记录领用到设备 | 操作简单 | 无法追踪具体部位和寿命 |
| C. 关键备件必须绑定，普通耗材可不绑定 | 按备件类型控制 | 平衡准确和效率 | 需要维护备件分类 |

推荐：C. 关键备件必须绑定，普通耗材可不绑定。

推荐原因：轴承、电机、传感器等关键件需要寿命追踪，螺丝、油脂等耗材不适合强制绑定。

### 4.3 采购申请是否进入首版

| 方案 | 说明 | 优点 | 风险 |
|------|------|------|------|
| A. 首版不做采购 | 只做库存和领用 | 简单 | 低库存无法闭环 |
| B. 轻量采购申请 | 低库存可发起采购申请，记录审批和到货 | 覆盖设备侧需求 | 不含完整采购执行 |
| C. 完整采购管理 | PR、PO、收货、对账 | 闭环完整 | 和 ERP 重叠，范围过大 |

推荐：B. 轻量采购申请。

推荐原因：设备管理需要表达“需要买什么”，但采购执行通常在 ERP。首版做申请和状态跟踪即可。
