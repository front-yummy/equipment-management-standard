## 产品化取舍原则

1. 主流程优先：先保证设备、维护、维修、备件、KPI、知识库能闭环。
2. 集成轻量化：外部库存、AI 服务只定义数据对接和失败兜底，不在 P9 内实现复杂平台能力。
3. 页面少而够用：每个模块只保留支撑主流程的列表、详情、配置、执行页面。
4. 状态少而清晰：不引入复杂审批、逐级上报、采购 PR/PO、完整 OEE 等长流程。
5. 异常可追溯：任务、工单、备件绑定、AI 推荐采纳都要记录操作人、时间和来源。

## 非 P9 范围

| 能力 | P9 处理方式 |
|------|-------------|
| 完整 OEE 填报、PPM、稼动率、FTY | 不纳入；P9 KPI 只保留 MTTR、MTBF、故障率 |
| 采购申请 PR、采购单 PO、PMC 调拨 | 不纳入；只展示库存和领用使用闭环 |
| 完整仓储、入库、出库、盘库 | 不纳入；库存以外部同步或手工初始化为准 |
| 复杂风控看板、呆滞库存、周转率 | 不纳入；只保留库存预警和寿命提醒 |
| 逐级上报、短信、邮件、企业 IM | 不纳入；P9 只要求系统待办和页面预警 |
| AI 自动制单、AI 自动提交审批 | 不纳入；AI 只做推荐和问答支撑，结果需人工确认 |
