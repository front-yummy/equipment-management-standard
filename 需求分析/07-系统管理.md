# 07. 系统管理

## 模块目标与边界

系统管理为设备管理系统提供登录、用户、角色、菜单、功能权限、数据权限、操作日志和系统通用配置能力。

本模块不维护业务主数据；组织、设备类型、备件分类、单位、供应商和业务字典归 `base 主数据` 模块。报表能力作为运营视图保留在本模块中，指标口径仍引用各业务模块。

用户、角色、菜单、权限等列表型维护页面需支持模板下载、导入、导出和错误报告。

## 报表范围

| 报表 | 数据来源 | 核心指标 |
|------|----------|----------|
| 设备报表 | 设备台账 | 设备数量、状态、类型、位置、变更记录 |
| 点检报表 | 点检任务 | 完成率、异常项、漏检项、逾期数 |
| 保养报表 | 保养任务 | 完成率、工时、备件消耗、逾期数 |
| 维修报表 | 维修工单 | 故障次数、停机时长、MTTR、MTBF、维修费用 |
| 备件报表 | 备件台账/领用/采购 | 库存周转、消耗量、采购趋势、水位风险 |
| OEE 报表 | OEE 损失和生产指标 | OEE、DT、DT率、PPM、FTY、损失分类 |


## 通用报表规则

1. 筛选条件包括时间范围、工厂、拉线、工段、工序、设备等。
2. 报表需遵守数据权限，用户只能看到权限范围内组织和设备。
3. 导出结果与当前筛选条件一致。
4. 指标口径应引用 OEE/KPI 模块，不在报表中重新定义。
5. 作废、取消、无效数据默认不进入统计，除非报表明确展示无效数据审计。

## 用户与角色

| 功能 | 规则 |
|------|------|
| 用户管理 | 用户名、工号、姓名、手机号、邮箱、部门、角色、启停 |
| 角色管理 | 角色名称、编码、描述、菜单权限、功能权限、数据范围 |
| 菜单管理 | 菜单树、路由、图标、排序、可见角色 |
| 操作日志 | 登录、权限变更、配置变更、关键操作追溯 |

## 权限规则

1. 菜单权限决定是否可进入页面。
2. 功能权限决定是否可新增、编辑、删除、导入、导出、审批、验收、派单等。
3. 数据权限决定可查看的工厂、拉线、部门、设备、仓库范围。
4. 移动端应复用同一权限模型。
5. 系统操作日志需记录关键配置变更。

## base 主数据引用

| 主数据 | 使用模块 |
|------|----------|
| 设备状态 | 设备台账、OEE、首页 |
| 损失/停机分类 | OEE 模块配置；维修、KPI 引用 |
| 备件分类 | 备件台账、看板 |
| 工单状态 | 维修、预防性维护 |
| 仓库/库位类型 | warehouse、备件管理 |

| 紧急程度 | 采购、调拨、维修 |

## 验收口径

1. 同一用户在 PC 与移动端权限一致。
2. 导出数据与页面筛选条件一致。
3. base 主数据停用后，新增业务不可再选择，但历史数据正常展示。
4. 菜单权限调整后重新登录生效，或按系统设计实时生效。
5. 用户、角色、菜单导入失败时可下载错误报告，错误需包含行号、字段和原因。

## 待澄清与迭代事项

1. 数据权限粒度是否到工厂、拉线、工序、设备、仓库，需要统一设计。
2. 报表是否需要定时邮件或企业 IM 推送，当前未纳入明确范围。
3. base 主数据变更是否需要审批或发布流程，建议由工作流模块配置。
